# Threat Literature Matrix Template

Use this file as `qa/threat_literature_matrix.md` when novelty, contribution, closest-threat, method-positioning, data-positioning, or literature-boundary claims enter prose.

## Purpose

Prevent novelty and contribution claims from outrunning the closest relevant literature.

## Threat matrix

| source_id | citation | relationship_to_project | threat_type | how_current_project_differs | claim_boundary_impact | source_verification_status | action |
|---|---|---|---|---|---|---|---|
| T001 |  | same question / same data / same method / adjacent setting / opposite finding | novelty / identification / data / method / interpretation |  |  | VERIFIED / ACCESS_LIMITED / UNSUPPORTED / CONTRADICTED / OPEN | cite / narrow claim / run robustness / drop novelty claim |

## Rules

- A novelty claim cannot be `VERIFIED` without checked closest-threat rows.
- "First", "new", "novel", "fills a gap", and "little is known" require explicit threat rows.
- If a threat paper uses the same data, same estimand, or same design family, the contribution claim must state the boundary precisely.
- If a threat source is access-limited, prose may not state strong novelty unless the limitation is visible.

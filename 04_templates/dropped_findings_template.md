# Dropped Findings

Purpose: record empirically observed but excluded results so writing cannot resurrect them without explicit clearance.

Rules:
- Start from rejected branches in `argument_decision.md`, pivot memos, audits, and review comments.
- Default ban location: Introduction, Discussion, Conclusion, Abstract.
- Reintroduction requires a later dated clearance entry with evidence.

| Finding / claim | Source table/figure/script | Why excluded | Banned from | Status | Clearance required |
|---|---|---|---|---|---|
|  |  | weak / off-argument / sample-specific / mechanical / not robust / unsupported | intro, discussion, conclusion, abstract | DROPPED |  |

## Notes

- Do not use dropped findings as teaser claims.
- Do not use dropped findings to inflate contribution.
- If a dropped finding is mentioned as a limitation or future direction, state explicitly that it is not part of the paper's evidence spine.
